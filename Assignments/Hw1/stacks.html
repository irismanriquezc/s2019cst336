Stacks are a type of container designed specifically to perform in a LIFO context( last- in first- out), where components are placed and extracted from only one end of the container.

Stack can be implemented quickly with an array or a linked list. Arrays are fast, but limited in size and Linked Lists involves overhead to be allocated, linked, unlinked and distributed, but not limited in size.

There are many real-life examples of a stack. Consider an example of plates stacked over one another in the canteen. The plate which is at the top is the first one to be removed, i.e. the plate which has been placed at the bottommost position remains in the stack for the longest period of time.

Some functions that stacks have are:
  • .pop(): pops the top element
  • .push(): pushes element into stack
  • .top(): accesses next element
  • .empty(): checks if stack is empty
  • .size(): returns the size of stack
